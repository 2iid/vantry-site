Why it exists

Claude Code is brilliant. Alone, it's one engineer with no process.

It forgets your standards, skips the tests, and asks "what's next?" after every task. ShipCrew gives it what a real team has: specialists that own their domain, skills with the checks baked in, and a delivery loop that never stalls — so the quality doesn't depend on re-deriving it every time.

Questions

Good to know

Does it work with my stack? ›
Yes. Core is stack-agnostic — the agents, skills, and delivery loop work in any language or framework. Pro adds a ready-made Next.js 15 + Supabase starter if you want a head start.
I already have a codebase — can I use this? ›
Yes — that's ShipCrew's superpower. Point /adopt at your existing repo (in place, a git URL, or a local folder) and it runs a deep, severity-ranked audit, writes a CLAUDE.md from your real code, and hands you a stabilize-first sprint plan + board. The audit is read-only; every fix lands as a PR.
Is this just a folder of prompts? ›
No. It's a system: agents with real judgment and clear ownership, skills with verification steps built in, a sprint board that runs itself, and security gates. The quality is in the process, not a single clever prompt.
How is it delivered? ›
After checkout you get a license key and access to a private GitHub repo. Clone it, or copy the .claude/ directory into your existing project — you're running in minutes.
Do I need Supabase or Next.js? ›
Not for Core. Those only matter for the optional Pro boilerplate. The agents and workflow are happy in a Python, Go, Rails, or plain-Node repo.
Updates and refunds? ›
Lifetime updates are included — buy once, keep getting improvements. If it's not for you, a 14-day no-questions refund has you covered.
